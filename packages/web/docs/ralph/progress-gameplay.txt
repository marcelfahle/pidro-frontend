Progress log for PRD gameplay implementation.

## 2026-03-08 — Phase 3.1: Categorized lobby sections

Implemented four categorized sections in the lobby page (LobbyPage.tsx):

1. **Your Tables** — Amber/orange styled cards for games the current user is in
   (status=playing + user in player_ids/seats). Shows "Rejoin" button that navigates
   directly to the game page (no API call needed).

2. **Open Tables** — Main section showing waiting rooms with "Join" button.
   Uses existing RoomTable component. Shows empty state when no rooms available.

3. **Need a Player** — Dashed-border cards for playing rooms with vacant seats.
   Shows "Substitute" button (calls joinRoom API).

4. **Watch** — Horizontal scrolling compact cards for spectatable games (playing,
   full seats, not the user's game). Shows "Watch" button.

Categorization is done client-side via `categorizeRooms()` in LobbyPage.tsx. The
server still sends a flat rooms array; if it ever sends categorized data, the
channel hook can be updated to handle it.

Also fixed:
- Updated RoomRow to use generic `onAction`/`actionLabel` interface instead of
  hardcoded Join-only props. Fixed error display bug (errors now persist after
  loading completes instead of never showing).
- Updated RoomTable to match new interface (actionLoadingCode, actionError).
- Fixed pre-existing lint errors in TrickArea.tsx (array index keys).
- Fixed all formatter issues across the codebase.
- Added 4 new tests for categorized sections (13 total lobby tests).
- Also marked PRD feature 3.2 (create table + online count) as passing since it
  was already fully implemented.

All checks pass: typecheck, lint, 111 tests.

## 2026-03-08 — Phase 6.2: Seat status indicators for disconnected/bot players

Implemented seat lifecycle event handling and visual indicators for disconnect
scenarios in the game table.

**Channel events added** (useGameChannel.ts):
- `player_reconnecting` — Sets seat to 'reconnecting' status, shows warning toast
- `bot_substitute_active` — Sets seat to 'bot_substitute', shows warning toast
- `player_reclaimed_seat` — Resets seat to 'normal', shows success toast
- `seat_permanently_botted` — Sets seat to 'permanent_bot', updates username to 'Bot'
- `substitute_joined` — Resets seat to 'normal' with new username, shows success toast

**Store changes** (shared/stores/game.ts):
- Added `seatStatus: SeatStatus` field to `PlayerMeta` interface
- Added `setSeatStatus(position, status, username?)` action
- `SeatStatus` flows through `RelativePlayerView` to the view model

**UI changes** (GamePlayerCard.tsx):
- Reconnecting state: amber spinner + "Reconnecting..." text, dimmed opacity
- Bot substitute state: robot emoji avatar, "Bot" badge (amber), subtext
  "Bot · [PlayerName] can rejoin"
- Permanent bot state: robot emoji avatar, "Bot" badge, simple "Bot" subtext
- DC badge only shown for plain disconnects (not reconnecting or bot states)

**Other changes:**
- Added `amber` variant to Badge component
- PlayerHand passes seatStatus through to GamePlayerCard
- GameTable passes seatStatus from viewModel to each PlayerHand
- GamePage passes onSeatEvent callback to useGameChannel for toast notifications
- Fixed pre-existing TrickArea.tsx typecheck error (trick.number → idx)
- Fixed pre-existing TrickArea.tsx lint warning (array index key → composite key)

**PRD updates:**
- Marked all previously-implemented features as passing (1.1-1.5, 2.1-2.2,
  4.1-4.4, 5.1-5.3, 6.1) — these were already built but the PRD wasn't updated
- Marked 6.2 as passing
