## 2026-03-08 — Phase 3.1: Categorized lobby sections

Implemented four categorized sections in the lobby page (LobbyPage.tsx):

1. **Your Tables** — Amber/orange styled cards for games the current user is in
   (status=playing + user in player_ids/seats). Shows "Rejoin" button that navigates
   directly to the game page (no API call needed).

2. **Open Tables** — Main section showing waiting rooms with "Join" button.
   Uses existing RoomTable component. Shows empty state when no rooms available.

3. **Need a Player** — Dashed-border cards for playing rooms with vacant seats.
   Shows "Substitute" button (calls joinRoom API).

4. **Watch** — Horizontal scrolling compact cards for spectatable games (playing,
   full seats, not the user's game). Shows "Watch" button.

Categorization is done client-side via `categorizeRooms()` in LobbyPage.tsx. The
server still sends a flat rooms array; if it ever sends categorized data, the
channel hook can be updated to handle it.

## 2026-03-08 — Phase 6.3: Owner decision banner for substitute seats

Implemented the owner decision banner that appears when a player disconnects and
the room owner needs to decide whether to open the seat for substitutes or keep
the bot.

**Channel event added** (useGameChannel.ts):
- `owner_decision_available` — Fires when a player disconnects and the server
  offers the room owner a choice. Calls new `onOwnerDecision` callback with
  `{ position, playerName }`.
- Added `OwnerDecisionEvent` type and `onOwnerDecision` option to hook interface.

**GamePage changes** (GamePage.tsx):
- Stores `hostId` in `roomConfigRef` from the Room API response.
- Maintains `ownerDecisionQueue` state (array of pending decisions) and
  `dismissedSeatsRef` (Set of positions already dismissed).
- `handleOwnerDecision`: only enqueues if current user is the room owner AND
  position hasn't been dismissed AND isn't already in the queue.
- `handleOpenSeat`: dismisses the position, removes from queue, pushes
  `open_seat` action to the game channel.
- `handleKeepBot`: dismisses the position and removes from queue (no server call).
- Banner is hidden during the owner's active turn (`isMyTurn` check) — queued
  events surface when the turn passes.
- Only one banner shown at a time (always `queue[0]`).

**New component** (OwnerDecisionBanner.tsx):
- Non-blocking amber banner positioned at the top of the game area (z-40).
- Shows "[PlayerName] left. Open their seat for another player?"
- Two buttons: "Open Seat" (amber solid) and "Keep Bot" (amber outline).
- Uses existing `animate-slide-in-top` CSS animation.

**Remaining features:** 7.1-7.3 (polish)

All checks pass: typecheck, lint, 111 tests.

## 2026-03-08 — Phase 7.2: Loading states and error handling throughout the app

Improved loading states, error handling, and added optimistic UI across the app.

**GamePage error improvements** (GamePage.tsx):
- Differentiated room errors by HTTP status: 404 → "Room not found. It may have
  been closed." / 403 → "You do not have access to this room." / other → "Failed
  to connect to server."
- Added "Retry" button to both room error and channel error states (alongside
  existing "Back to Lobby" button). Retry re-fetches the room and reconnects
  the channel. Room-not-found errors only show "Back to Lobby" (no retry).
- Extracted `fetchRoom()` as a reusable callback for both initial load and retry.
  Uses a `fetchIdRef` counter to cancel stale fetches.

**Optimistic card play** (GamePage.tsx, GameTable.tsx, TrickArea.tsx):
- When a player clicks a card to play it, the card immediately moves from their
  hand to the trick area (south slot) before the server responds.
- `optimisticCard` state in GamePage flows to GameTable → TrickArea.
- GameTable filters the optimistic card from the south player's hand so it
  disappears instantly.
- TrickArea adds the optimistic card to the south trick slot if not already
  present from server state.
- Cleared automatically when server state updates (confirming the play).
- On error: cleared immediately, card reappears in hand with shake animation
  and error toast.

**Lobby channel error** (useLobbyChannel.ts, LobbyPage.tsx):
- Lobby channel join failures now call `setError()` on the lobby store (previously
  only logged to console).
- LobbyPage reads the error state and shows a red error banner with "Retry" button
  when the lobby channel fails to connect.

**Test updates:**
- Updated GamePage test to match new error message text and verify Retry button.

**Remaining features:** 7.1 (responsive layout), 7.3 (turn indicators/game feel)

All checks pass: typecheck, lint, 111 tests.
